## 2. Model purpose & use

- **Purpose.** Estimate the probability that a loan defaults within 12 months.
- **Intended use.** (a) risk-based approve/decline/price banding; (b) the PD
  input to the IFRS 9 ECL engine (`ECL = PD × LGD × EAD × DF`); (c) Stage 1/2
  determination via the 12m-PD SICR threshold (0.15).
- **Users.** Credit underwriting, provisioning/finance, portfolio monitoring.
- **Materiality.** Tier 1 — PD directly drives the reported allowance
  ($211.2m weighted ECL, 12.45% coverage on the reference run) and staging split.
- **Out of scope.** LGD, EAD, macro-scenario design, and lifetime-PD extrapolation
  are separate components (validated/summarised in §11).

## 4. Conceptual soundness

- **Method fit.** Logistic regression is an industry-standard, transparent PD
  method; coefficients are directly interpretable and the model is monotonic in
  each standardised driver. Appropriate for a regulated Tier 1 use where
  explainability and calibration matter more than marginal AUC.
- **Feature set.** Drivers (credit score, DTI, LTV, DSCR, utilisation, missed
  payments, max DPD, sector, region) are economically sensible and align with
  the latent data-generating process — signs are as expected (risk rises with
  lower score, higher DTI/LTV, lower DSCR).
- **Calibration layer.** Isotonic calibration wraps the classifier so outputs are
  usable as **probabilities**, not just rankings — essential because PD feeds ECL
  directly. This is the correct design choice.
- **Challenger.** Gradient boosting was fit as a challenger (§6); the simpler,
  calibrated logistic model was retained on the discrimination/robustness
  trade-off.

**Assessment: Satisfactory.**

## 5. Performance validation

### 5.1 Discrimination (held-out 30% test set)

| Model | ROC-AUC | PR-AUC | Brier |
|-------|--------:|-------:|------:|
| **Logistic regression (champion)** | **0.9407** | **0.8022** | **0.0718** |
| Gradient boosting (challenger) | 0.9356 | 0.7917 | 0.0743 |

- ROC-AUC 0.94 indicates strong rank-ordering. PR-AUC 0.80 confirms performance
  holds on the minority (default) class, the metric that matters at ~18% base rate.
- **Caveat:** metrics reflect a *designed* synthetic signal; on real data expect
  lower AUC and noisier calibration (see §8).

### 5.2 Calibration & risk bands

Observed default rate rises monotonically across score bands — the primary
calibration test for a banded PD model:

| Band | n | % of book | Avg predicted PD | Observed default rate |
|------|---:|----------:|-----------------:|----------------------:|
| A (Prime) | 1,715 | 47.6% | 0.51% | 0.64% |
| B (Near-prime) | 437 | 12.1% | 4.68% | 2.75% |
| C (Acceptable) | 270 | 7.5% | 10.71% | 12.22% |
| D (Watch) | 363 | 10.1% | 20.32% | 22.31% |
| E (Substandard) | 815 | 22.6% | 66.86% | 64.91% |

- Ranking is **strictly monotonic** and the extreme bands (A, E) are well
  calibrated. **Finding F1:** Band B over-predicts (4.68% vs 2.75% observed) and
  Band C slightly under-predicts — a mid-range calibration wobble typical of
  isotonic fits on thin bands. Immaterial to total ECL but should be monitored.
- Brier score 0.072 confirms good aggregate probability accuracy.

### 5.3 Threshold / policy performance

At the 0.15 policy cut (also the Stage 2 SICR threshold), a confusion matrix is
produced (`reports/figures/pd_confusion.png`). Validation confirms the threshold
is consistent between the PD policy band boundary and the ECL staging rule —
**no cross-component inconsistency**.

## 7. Population stability (PSI) & sensitivity

### 7.1 Population Stability Index

PSI compares score distributions across samples; the standard rule is
**PSI < 0.10 = stable, 0.10–0.25 = moderate shift, > 0.25 = significant shift**.

- **Train vs test PD distribution: PSI ≈ 0.02 (stable)** — expected given a random
  split; confirms no split-induced distortion.
- The monitoring plan (§9) operationalises PSI on **score bands** and **key input
  features** (credit score, LTV, DTI) against the development baseline each
  quarter. A reference implementation is provided below so the check is runnable.

```sql
-- PSI of the PD score-band distribution: monitoring period vs baseline.
-- PSI = Σ (actual% − expected%) * ln(actual% / expected%)
WITH base AS (   -- expected: development sample share per band
    SELECT band, COUNT(*)::NUMERIC / SUM(COUNT(*)) OVER () AS expected_pct
    FROM pd_scored_baseline GROUP BY band
),
curr AS (        -- actual: current monitoring sample share per band
    SELECT band, COUNT(*)::NUMERIC / SUM(COUNT(*)) OVER () AS actual_pct
    FROM pd_scored_current  GROUP BY band
)
SELECT ROUND(SUM((c.actual_pct - b.expected_pct)
                 * LN(c.actual_pct / b.expected_pct)), 4) AS psi
FROM base b JOIN curr c USING (band);
```

### 7.2 Sensitivity testing

The ECL/stress layer applies PD multipliers (base 1.00 / downside 1.75 / severe
3.00). Validation confirms PD flows through sensibly:

| Single-factor shock | Portfolio expected loss | Δ vs base | % change |
|---------------------|------------------------:|----------:|---------:|
| PD +25% (unemployment proxy) | $143.7m | +$22.4m | +18.4% |
| PD +50% (unemployment proxy) | $157.8m | +$36.4m | +30.0% |
| PD +25% & LGD +15% (combined) | $165.3m | +$43.9m | +36.2% |

Loss responds monotonically and proportionately to PD shocks with no
discontinuities — behaviour is stable and explainable.

## 8. Backtesting

- **In-sample split backtest.** Predicted vs observed default rates by band (§5.2)
  constitute the primary backtest: 4 of 5 bands within tolerance, 1 flagged (F1).
- **Aggregate.** Predicted portfolio default rate ≈ observed (18.49%); no systemic
  over-/under-prediction at the total level.
- **Gap (F3).** No out-of-time backtest is possible on a single cross-section.
  Recommended once temporal data exists: rolling 12-month OOT windows with a
  **Hosmer–Lemeshow** calibration test and a **binomial/traffic-light** test per
  band (green/amber/red on observed-vs-expected default counts).

## 10. Recommended controls

1. **Champion/challenger discipline** — re-run the GBM challenger at each
   revalidation; retain champion only if it remains competitive on AUC *and*
   calibration.
2. **Calibration guardrail** — recalibrate (isotonic/Platt) before deployment if
   any band breaches the §9 calibration trigger (addresses F1).
3. **Assumption change control** — staging thresholds, score bands, and scenario
   multipliers are centralised in `config.py`; any change requires MRM sign-off
   and a rerun of the validation pack.
4. **Reproducibility control** — fixed seed + committed test suite; validation is
   fully re-performable via `python run_pipeline.py && pytest`.
5. **Use-restriction control** — model outputs flagged as synthetic/demonstration;
   not permitted for production underwriting until real-data revalidation.

## 12. Limitations & conclusion

**Limitations.** (1) Synthetic data with a designed signal — real-world
performance will be lower and messier. (2) Point-in-time 12m PD with no term
structure (F2). (3) No out-of-time validation (F3). (4) Mid-band calibration
wobble (F1).

**Conclusion.** The PD model is **conceptually sound, well-discriminating, and
adequately calibrated** for its intended (demonstration) use, and is **approved
subject to the three findings and the monitoring plan**. The same validation
framework — data quality, conceptual soundness, discrimination, calibration,
PSI, sensitivity, backtesting, challenger, monitoring, controls — is directly
transferable to a real portfolio once real, time-series data is available.
